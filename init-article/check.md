# 演讲稿交付自检清单

撰写完成后，逐项核对。**全部通过**方可输出 `speech.txt`。

## 硬性指标

- [ ] 正文字数 ≤ **900 字**（不含 PPT 标记行；用工具或人工计数）
- [ ] 按 300 字/分钟估算，朗读时长 ≤ **3 分钟**
- [ ] 文件输出路径：项目根目录 `speech.txt`

## 内容完整性

- [ ] 提到项目名称（小说提示词 AI 生成器 / Novel Prompt AI）
- [ ] 说明核心痛点（提示词耗时 / 云端隐私 / 缺乏结构化）
- [ ] 说明解决方案（本地 LLM + 四阶段流水线）
- [ ] 提到至少 2 个手写数据结构（HashMap / LinkedList / Queue）
- [ ] 引用至少 2 个报告中的实测数据（耗时、字数、效率、成功率等）
- [ ] 有开场问候与结尾致谢

## 数据准确性（对照 report.md）

- [ ] 端到端耗时：1～3 分钟（或 213～255 秒）
- [ ] 输出规模：约 2000 字
- [ ] 人工对照：15～20 分钟
- [ ] 效率提升：6～8 倍（或 4.4～4.7 倍，若用对照组数据）
- [ ] 成功率：90%（10 次中 9 次）
- [ ] 未编造报告中不存在的数据

## 表达质量（对照 voice.md）

- [ ] 口语化，适合朗读
- [ ] 无过度学术腔或营销腔
- [ ] 关键术语首次出现可读
- [ ] 短句为主，无明显超长句

## PPT 切分（提示词 4 阶段，若需）

- [ ] 文件开头有：`第一页PPT：封面`
- [ ] 约每 100 字插入：`更换下一页PPT：{标题}`
- [ ] 文件末尾有：`最后一页PPT：致谢`

## 最终动作

1. 通读全文朗读一遍（或默读计时）
2. 修正拗口处
3. 确认字数后写入 `speech.txt`
