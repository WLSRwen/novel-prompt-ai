# 演讲主题与核心信息

## 演讲定位

- **类型**：课程项目答辩 / 3 分钟汇报演讲
- **项目**：Novel Prompt AI（小说提示词 AI 生成器）
- **汇报人**：文冬来
- **核心命题**：如何用「手写数据结构 + 本地 LLM + 轻量 GUI」实现从一句话创意到结构化长提示词的端到端生成

## 必须传达的信息（按优先级）

1. **问题**：网文作者写提示词耗时长、云端工具有隐私与成本顾虑、通用对话无法输出固定 JSON
2. **方案**：桌面端 Tkinter 应用，LM Studio 本地调用 Qwen3.5-9B，四阶段流水线（解析→设定→剧情→组装）
3. **亮点**：自主实现 HashMap、LinkedList、Queue；队列 FIFO 调度；数据结构与业务深度结合
4. **效果**：端到端 1～3 分钟，输出约 2000 字结构化提示词；人工撰写约 15～20 分钟，效率提升约 6～8 倍；10 次试跑成功率 90%
5. **价值**：兼顾实用（本地隐私）与教学（可演示、可答辩）

## 可选提及（篇幅允许时）

- 三层/四层架构：表示层、业务层、数据结构层、模型层
- JSON 三层保障：提示层 + 传输层 + 程序层
- 未来：异步推理、LLM 剧情生成、PyInstaller 打包

## 禁止事项

- 不编造报告中未出现的数据
- 不展开冗长技术细节（如 rehash 算法、具体 prompt 全文）
- 不使用英文缩写而不解释（LLM 首次出现可简注「大语言模型」）
